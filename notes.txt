# Kimis System - Technical Documentation

## 1. Overview
KiMiS (Knowledge Intelligence Management System) is a modular web application designed for workspace collaboration, knowledge management, and visual process mapping.

The system is built with a **Node.js/Express** backend and a **Vanilla HTML/CSS/JS** frontend. It follows a modular architecture where features are separated into dedicated route and module files.

## 2. Architecture & File Structure
The project has been refactored from a monolithic script into a clean, modular structure:
- **`server.js`**: Slim entry point that bootstraps the Express app.
- **`config/db.js`**: Manages the PostgreSQL connection pool.
- **`middleware/`**: Contains `auth.js` (JWT verification) and `rbac.js` (Workspace-level permissions).
- **`routes/`**: Feature-specific API endpoints (`auth`, `users`, `workspaces`, `members`, `articles`, `workflows`).
- **`frontend/`**: Separates the UI into HTML, `css/dashboard.css`, and modular JS files in `js/`.

## 3. Database Schema (PostgreSQL)
The system uses a relational PostgreSQL database with the following core tables:
- **`users`**: Auth data (email, password) and global roles (admin/user).
- **`workspaces`**: Logical containers for teams and projects.
- **`workspace_memberships`**: Links users to workspaces with specific roles.
- **`articles`**: Knowledge base entries (Markdown supported).
- **`workflows`**: Metadata for processes.
- **`workflow_nodes`**: Steps in a process (action, decision, start, end, etc.).
- **`workflow_edges`**: Directional connections between nodes with optional conditions.

## 4. Authentication & Security
- **JWT (JSON Web Tokens)**: Stateless authentication. Tokens are stored in the browser's `localStorage` or `sessionStorage`.
- **Stateless Validation**: Backend verifies the token signature on every protected request.
- **Prepared Statements**: All database queries use parameterization (e.g., `$1, $2`) to prevent SQL Injection.
- **Security Middleware**: `authenticateToken` ensures the user is logged in before accessing any feature.

## 5. Role-Based Access Control (RBAC)
KiMiS implements a dual-layer RBAC system:
### Layer 1: Global Roles
- **Admin**: Can access system-wide sensitive data and manage global users.
- **User**: Standard access.

### Layer 2: Workspace Roles
Permission is checked per workspace using the `checkWorkspaceRole` middleware:
- **Owner**: Full control over the workspace, articles, and workflows. Can delete the workspace and manage all members.
- **Admin**: Can manage most members, articles, and workflows.
- **Editor**: Can create and edit articles/workflows but cannot manage workspace membership.
- **Viewer**: Read-only access to articles and workflows within the workspace.

## 6. Knowledge Articles (KB)
The KB module allows users to document procedures and information:
- **Markdown Support**: Uses `marked.js` on the frontend for rich text rendering.
- **Versioning**: Track updates through `created_at` and `updated_at`.
- **Drafts/Archiving**: Articles can be "deleted" (soft-delete/archive) without removing them from the DB.
- **Linkage**: Articles can be linked to specific nodes in a visual workflow for contextual documentation.

## 7. Visual Workflow Engine (Graph-Based)
Unlike a simple list, KiMiS uses a directed graph to represent processes:
- **Nodes**: Represent actions, decisions, start/end points, and notes.
- **Edges**: Directional arrows connecting nodes.
- **Conditional Branching**: Edges can carry "conditions" (e.g., "Yes/No") to represent decision paths in a process.
- **Referential Integrity**: Deleting a workflow cascade-deletes its nodes and edges. Deleting a node removes its inbound and outbound edges.
- **Visual Persistence**: Node coordinates (`position_x`, `position_y`) are saved to allow a persistent visual layout on the canvas.

## 8. API Endpoints Summary
- **/api/auth**: `/login`, `/register`, `/health`
- **/api/me**: Profile management and global admin data.
- **/api/workspaces**: CRUD for workspaces.
- **/api/workspaces/:wsId/members**: Manage team members and roles.
- **/api/workspaces/:wsId/articles**: KB management.
- **/api/workspaces/:wsId/workflows**: Visual workflow management including nodes and edges.
